Discussion Post: Review of IT Code of Conduct

Review of IT Code of Conduct

Key Documents

The document "Communication: Data protection rules as a trust-enabler in the EU and beyond – taking stock" (European Commission, 2019) talks about the critical role of data protection in forming trust among individuals, businesses and public authorities. The General Data Protection Regulation (GDPR, 2016) outlines data protection laws for the European Union. It establishes the right of individuals regarding personal data. It should be noted that UK organisations no longer need to comply with these documents but with the UK versions of them.

I reviewed the established Code of Conduct for a Sixth Form College in the UK. There is a core Code of Conduct document that is supported by individual documents covering, such as the Internet Acceptable Use Policy, the Setting Passwords Protocol, and the BYOD to Work Policy.

Best Practices

The Data Protection Policy follows the GDPR(UK) and lays out the responsibilities for protecting personal data. A lot of student and teacher data is held, and this document is clear but comprehensive, including procedures for data access requests and a data retention schedule. The Internet Acceptable Use Policy defines what is reasonable behaviour and what actions could compromise the network. This is an important document as it not only applies to teachers but also to students.

Areas for Improvement

When conducting this research, I found that each of the policies and procedures were b but found it difficult to locate them. They are all stored in one folder with all the other college policy documents. A unified Code of Conduct document could improve comprehension of the subjects and therefore compliance. However, retaining separate documents can allow for more detail to be included for each.

Incident Reflection

Recently the sixth form conducted a simulated phishing attack to see the actions of staff. Many staff opened the email and read it, and many opened the attachment that asked for a username and password. The exercise highlighted the need for ongoing staff education and clear reporting protocols.

Future Implications

The sixth form is currently going through the process of gaining Cyber Essentials Certification (National Cyber Security Centre, N.D.) and are currently in the data collection phase. We could also establish cyber security training for new and current staff to highlight the documents and where to find them, giving key details. There was also no formal incident response plan except to tell IT support.

An example of an organisation's IT Code of Conduct can be seen in the policies of Amazon. Amazon places a strong emphasis on data protection and compliance with regulations such as the General Data Protection Regulation (GDPR, 2018). Their IT Code of Conduct outlines specific data handling practices that prioritise accountability, transparency, and lawful data processing (Vander Maelen, 2023). For instance, employees are required to follow stringent guidelines on collecting, using, storing, and deleting personal data, ensuring that only data necessary for business purposes is retained, in line with GDPR’s principles of data minimisation (European Commission, 2019).

Furthermore, Amazon restricts access to personal data to authorised personnel based on job responsibilities, enforcing secure login protocols that include multi-factor authentication to prevent unauthorised access. The organisation also mandates encryption of all personal data, both in transit and at rest, to protect data integrity and confidentiality (Delgado-von-Eitzen et al., 2021). In case of any data breaches or security incidents, employees are instructed to report them immediately to the IT department, adhering to GDPR’s requirement for timely breach notifications.

Moreover, Amazon conducts regular training sessions to educate employees about data protection laws, cyber threats, and secure data handling practices, reinforcing their roles in maintaining compliance (British Computing Society, 2022). By implementing such comprehensive measures, Amazon demonstrates its commitment to data security and user trust, effectively aligning its IT practices with GDPR standards.

Your fellow colleague pursuing an MSc in Computer Science and currently working as a Software Engineer at Digital Processing Systems (DPS). My role involves developing robust solutions for automating repetitive tasks and implementing data ETL methodologies in automation processes. In this post, I reflect on Data Protection and IT Code of Conduct, focusing on how DPS aligns with GDPR principles. By examining DPS’s IT policies, I will highlight best practices, identify areas for improvement, and share lessons learned from real-world incidents involving data protection.

Best Practices The Information Security Policy of DPS exemplifies several GDPR-compliant practices:

  1. Confidentiality and Access Control: Information is protected against unauthorized access, ensuring confidentiality (DPS, 2015).
  2. Incident Reporting: Breaches and suspected vulnerabilities are promptly reported and investigated, aligning with GDPR Article 33.
  3. Risk Assessment: Regular risk assessments identify and mitigate potential security failures, supporting GDPR’s accountability principle.
Additionally, the IT Service Management Policy emphasizes service continuity and availability management, which is critical for maintaining data availability as mandated by GDPR Article 32.

Areas for Improvement While DPS has a robust policy framework, the following areas could be enhanced:

  1. Employee Training: Training programs on GDPR compliance are not explicitly outlined. Frequent, targeted training can reinforce the importance of protecting personal and sensitive data.
  2. Data Retention Clarity: The retention policy for sensitive information needs greater transparency to comply with GDPR's storage limitation principle (Regulation (EU) 2016/679, Article 5).
  3. Vendor Oversight: Although secure information sharing is highlighted, stronger contractual obligations for third-party compliance could minimize risks.

Incident Reflection An incident occurred involving unauthorized access to sensitive client information due to inadequate access control policies. The specific scenario involved files from a client being scanned by designated personnel at DPS. After scanning, the files were stored on a shared storage system accessible by the scanning team, which posed a risk of unauthorized access.

Fortunately, the issue was mitigated before any harm occurred. As part of the mitigation process, I contributed by:

  1. Developing automated scripts to move files from a folder accessible by the scanning team to a highly restricted folder. These scripts ensured secure and complete transfer while eliminating traces from the original folder.
  2. Adding a User Access Control (UAC) prompt for accessing the restricted folder, requiring specific credentials.
  3. Implementing Active Directory Policies to restrict folder access to a pre-approved list of users as agreed upon with the client.

Recommendations To prevent similar incidents and strengthen overall compliance:

  1. Conduct regular training sessions for employees on GDPR compliance and emerging data security threats.
  2. Enforce a clear and transparent data retention policy to avoid unnecessary storage of sensitive information.
  3. Integrate Data Loss Prevention (DLP) tools to monitor and secure sensitive data transfers.
  4. Strengthen vendor contracts with specific clauses to ensure third-party compliance with GDPR principles.
DPS's policies demonstrate a commitment to data security, reflecting GDPR principles and fostering trust. By addressing the identified gaps and adopting a proactive approach, DPS can further enhance its compliance framework and ethical practices in computing.

Wiki Activity: Website Functionality

The process of requesting and displaying a web page begins when a user inputs a Uniform Resource Locator (URL) into their web browser. Since the internet operates using numerical IP addresses rather than human readable domain names, the browser must first perform Domain Name System (DNS) resolution.

The browser then queries a DNS server to translate the domain name into its corresponding IP address. Once the IP address is obtained, the browser initiates a Hypertext Transfer Protocol (HTTP) request to the web server at that address. This request traverses the internet infrastructure through multiple routers using TCP/IP protocols until it reaches the destination server. The server processes it and responds by transmitting the requested web page files.
