Website Functionality

The process of requesting and displaying a web page begins when a user inputs a Uniform Resource Locator (URL) into their web browser. Since the internet operates using numerical IP addresses rather than human readable domain names, the browser must first perform Domain Name System (DNS) resolution.

The browser then queries a DNS server to translate the domain name into its corresponding IP address. Once the IP address is obtained, the browser initiates a Hypertext Transfer Protocol (HTTP) request to the web server at that address. This request traverses the internet infrastructure through multiple routers using TCP/IP protocols until it reaches the destination server. The server processes it and responds by transmitting the requested web page files.

Emerging Technologies and Network

Key Terms

Internet of Things: The Internet of Things refers to the network of physical objects that are embedded with sensors, software and other technologies to connect and exchange data with other devices and systems over the internet.

Edge Computing: Edge computing is a distributed computing framework where data processing is done closer to the data source rather than relying solely on a central server. This reduces latency, conserves bandwidth, and enhances real time decision making.

Artificial Intelligence (AI): Artificial Intelligence involves the development of algorithms and systems that can simulate human intelligence. This includes machine learning, natural language processing, computer vision and robotics.

Case Study

A Survey on the Convergence of Edge Computing and AI for UAVs: Opportunities and Challenges (McEnroe et al. 2022)

The case study examines the combination of Edge Computing and AI in Unmanned Aerial Vehicles (UAVs) or drones. This integration enables real time data processing for faster decision making and improved autonomous flight capabilities. AI allows drones to comprehend complex environments while edge computing minimises latency by processing data closer to the source. This ensures quick responses during critical operations.

The technological integration enhances drone performance in surveillance, search and rescue operations and environmental monitoring. Edge computing lets drones analyse sensor data locally on the device. This reduces dependence on remote servers and improves overall response times. The approach marks a significant advancement in drone technology by merging AI's analytical capabilities with edge computing's efficiency to create more responsive unmanned aerial systems.

Issues and Challenges

  • Limited Computational resources

UAVs are typically constrained by size and weight, which limits their computational power. Running AI models locally on drones requires significant processing capability.

  • Energy Consumption

AI algorithms, especially deep learning models, can be computationally expensive and demand a lot of power.

  • Security and Privacy Concerns

UAVs are vulnerable to cyberattacks. Since they may collect sensitive data, such as surveillance footage or environmental data, ensuring the security and privacy of this information becomes crucial.

  • Data Overload

UAVs generate large amounts of data from sensors and cameras. Efficient data management and processing are needed to avoid overwhelming the system, especially in real-time situations.

  • Regulatory and Ethical Issues

The use of UAVs with AI and edge computing might face legal and ethical barriers, especially regarding privacy rights and airspace regulations.

  • Scalability and Deployment

Managing numerous intelligent drones requires robust systems for updates, maintenance and deployment across diverse environments.

The case study demonstrates how edge computing and AI integration enhances UAV capabilities through improved autonomy and real time processing. Multiple challenges impact the widespread adoption of this technology including resource constraints and energy requirements. Security concerns and legal considerations must also be addressed to ensure reliable system deployment.

Data Structures

Compare and Contrast: Linear and Non-Linear data structures

Linear Data Structures:

In linear data structures elements are arranged in a sequential manner. Each element is connected to its previous and next element. Access to elements follows a strict order, such as the first element being accessed first (FIFO) or the last element being accessed first (LIFO).

Examples: Arrays, Linked Lists, Stacks, and Queues.

Characteristics:

  • Sequential: Elements are stored in a linear sequence.
  • Memory Allocation: Typically contiguous in memory (arrays) or linked (linked lists).
  • Operations: Generally support operations like traversal, insertion, and deletion at known locations.

Non-Linear Data Structures:

In non-linear data structures, elements are not arranged sequentially. Instead, elements can have multiple connections, allowing for a hierarchical or network structure. These structures allow more complex relationships between elements.

Examples: Trees, Graphs.

Characteristics:

  • Non-sequential: Elements can have multiple connections and are not in a fixed order.
  • Memory Allocation: May require pointers (graphs and trees) or other referencing mechanisms.
  • Operations: Operations can be more complex, often involving searching, traversal (depth-first or breadth-first), and manipulation.

Compare and Contrast: Stacks and Queues

Stacks:

A stack is a linear data structure that follows the Last In, First Out (LIFO) principle, meaning the last element added to the stack is the first one to be removed. It operates much like a stack of plates, where you can only add or remove plates from the top.

Key Operations:

  • Push: Adds an element to the top of the stack.
  • Pop: Removes the top element.
  • Peek: Returns the top element without removing it.
  • IsEmpty: Checks if the stack is empty.

Applications:

  • Undo/Redo operations in software.
  • Expression evaluation
  • Function call management

Queues

A queue is a linear data structure that follows the First In, First Out (FIFO) principle, meaning the first element added is the first one to be removed. It operates like a queue of people where the first person in line is the first to be served.

Key Operations:

  • Enqueue: Adds an element to the rear of the queue.
  • Dequeue: Removes an element from the front of the queue.
  • Peek: Returns the front element without removing it.
  • IsEmpty: Checks if the queue is empty.

Applications:

  • Scheduling tasks in operating systems.
  • Handling requests in web servers.
  • Data transmission

Data Sort Using Cards

Suppose you are sorting a set of cards. The right hand holds several cards which are sequentially sorted by numbers arranged from right to left, say for example, five cards with 10, 6, 5, 3, 2 arranged from right to left. Your left hand is picking up the top card from a pile of card deck. Now you insert the card that you have picked from the pile of deck into the right position within the right-hand cards, in a sorted order from right to left.

Write the instructions in a step-by-step manner. Note down even a minor step that you have taken to insert the left-hand card into a correct position within the set of right-hand cards so that the right-hand card set remains sorted from right to left.

Answer:

  • Initial Setup: The right hand holds a sorted set of cards (10, 6, 5, 3, 2), and the left hand holds a pile of unsorted cards.
  • Pick a Card: The top card from the left hand pile is picked.
  • Compare: Starting from the rightmost card in the right hand set, the picked card is compared with the cards in the right hand set.
  • Shift Cards: If the picked card is larger than a card in the right hand set, the card is shifted one position to the left.
  • Insert: The picked card is placed in the correct position where it is no longer larger than the next card in the right hand set.
  • Repeat: The process is repeated for each new card from the left hand pile until the right hand set is fully sorted.

Insertion Sort

Insertion sort works by building a sorted array one element at a time. The algorithm splits a collection of items into two parts: a sorted portion and an unsorted portion.

Starting with the first element marked as sorted, the algorithm progressively takes each unsorted element and places it into its correct position within the sorted portion. To find this position, it compares the current element with each element in the sorted portion, moving sorted elements one position ahead to make space for the new element when necessary.

The process continues until all elements have been sorted. This algorithm is particularly efficient for small data sets or nearly sorted arrays, with a time complexity of O(n²) in the average and worst cases, but O(n) in the best case when the array is already sorted.
